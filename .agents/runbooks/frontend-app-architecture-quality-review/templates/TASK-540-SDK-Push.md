# 消息推送 SDK（Google Firebase Push、极光等）

> updated_by: Cascade - Cascade
> updated_at: 2026-05-15 16:56:00

- **消息推送 SDK（Google Firebase Push、极光等）**：
  - 推送能力是否通过统一抽象提供：业务是否依赖自有推送接口，避免直耦合 Firebase、极光等具体厂商 API
  - 厂商通道与环境隔离是否清晰：Google Firebase Cloud Messaging、极光及系统厂商通道的启用条件、优先级、降级策略、测试/生产配置是否可治理
  - Token 与设备绑定是否可靠：Token 获取、刷新、上报、解绑、用户切换、多设备、多账号、注销与重装场景是否闭环
  - 通知权限与合规是否可控：权限申请时机、拒绝授权降级、隐私同意前停用、消息订阅/退订与用户偏好是否一致
  - 消息路由与安全是否明确：通知消息/透传消息/站内信的边界、Deep Link 白名单、参数校验、敏感信息保护与重放/伪造风险是否处理
  - 到达率与可靠性是否可观测：发送、厂商回执、客户端接收、点击、失败原因、重试、频控、静默失败与告警是否形成闭环
  - 前后台与生命周期处理是否稳定：冷启动拉起、前台展示、后台限制、进程被杀、系统省电策略、通知渠道与角标等平台差异是否有兼容策略
